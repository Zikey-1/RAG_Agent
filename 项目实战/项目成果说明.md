# AI 驱动的智能客服知识库系统

## 项目概述

本项目构建了一个基于 **RAG（Retrieval-Augmented Generation）架构** 的智能客服系统，融合了向量检索、长链推理和对话记忆等核心能力，为用户提供精准、专业的个性化问答服务。

---

## 一、项目解决的核心痛点

### 1. 传统问答系统的知识局限
- **问题**：传统聊天机器人依赖预训练模型的内置知识，无法处理领域特定知识（如产品手册、内部文档等）
- **解决方案**：通过向量数据库持久化存储领域知识，实现实时知识检索与更新

### 2. 知识重复加载问题
- **问题**：文件上传时无法识别重复内容，导致向量数据库冗余膨胀
- **解决方案**：引入 MD5 校验机制，自动检测重复内容并跳过重复上传

### 3. 对话上下文丢失
- **问题**：普通问答系统无法记忆历史对话，每次交互都是独立的
- **解决方案**：集成 `RunnableWithMessageHistory`，实现多轮对话上下文保持

### 4. 长文本处理困难
- **问题**：长文档直接输入模型会超出 Token 限制
- **解决方案**：使用 `RecursiveCharacterTextSplitter` 进行智能文本分割

## 二、核心逻辑流

### 系统架构图

```
用户提问 → 检索器(Retriever) → 向量数据库(Chroma) → 相关文档
                                                         ↓
用户提问 + 参考资料 + 对话历史 → 提示词模板(ChatPromptTemplate) → 大模型(ChatTongyi) → 回答
```

### 关键技术组件

#### 1. 向量存储服务 (`vector_store.py`)
- **功能**：基于 Chroma 构建向量数据库，实现文本向量化存储和相似度检索
- **核心代码**：
```python
self.vector_store = Chroma(
    collection_name=config.collection_name,
    embedding_function=self.embedding,
    persist_directory=config.persist_directory
)
```

#### 2. 知识库管理 (`knowledge_base.py`)
- **功能**：处理文件上传、文本分割、MD5 去重、向量入库
- **核心能力**：
  - 自动检测重复内容（MD5 校验）
  - 智能文本分割（支持长文档处理）
  - 元数据记录（来源、时间戳）

#### 3. RAG 服务 (`rag.py`)
- **功能**：构建完整的检索增强生成链
- **核心流程**：
  1. **输入解析**：提取用户问题
  2. **向量检索**：从数据库中查找相关文档
  3. **提示词构建**：组合上下文、历史记录、用户问题
  4. **模型推理**：调用通义千问模型生成回答
  5. **流式输出**：支持实时响应

#### 4. 智能体开发 (`02Agent的stream的流失输出.py`)
- **功能**：构建工具调用型智能体
- **核心能力**：
  - 自定义工具注册（如股价查询、股票信息查询）
  - 工具调用决策逻辑
  - 流式输出思考过程

## 三、技术亮点

### 1. 长链推理能力
- 通过 `RunnableLambda` 实现复杂的链式处理
- 支持多步骤数据转换和处理
- 结合 `RunnablePassthrough` 实现并行计算

### 2. 多工具协作（Agent 模式）
- 支持自定义工具注册
- 自动决策何时调用工具
- 工具调用过程透明化（可追踪思考过程）

### 3. 数据持久化与增量更新
- Chroma 向量数据库本地持久化
- MD5 校验实现增量更新
- 支持动态知识扩展

### 4. 流式输出体验
- 支持实时响应，提升用户体验
- 通过 `write_stream` 实现逐字输出效果

## 四、应用场景

### 场景一：智能尺码推荐
- **输入**：用户提供身高、体重等信息
- **处理**：检索知识库中的尺码表数据
- **输出**：精准的个性化尺码推荐

### 场景二：股票信息查询（Agent 模式）
- **输入**："传智教育股价多少，并介绍一下"
- **处理**：
  1. 识别需要调用工具
  2. 调用 `get_price` 获取股价
  3. 调用 `get_info` 获取公司信息
  4. 整合结果生成回答

### 场景三：产品文档问答
- **输入**：关于扫地机器人的故障排查问题
- **处理**：检索维护保养文档和故障排除指南
- **输出**：专业的故障诊断和解决方案

## 五、部署与运行

### 启动方式

```bash
# 启动智能客服界面
streamlit run app_qa.py

# 启动文件上传界面
streamlit run app_file_uploader.py
```

### 配置说明

- **嵌入模型**：`text-embedding-v4`（阿里云通义千问）
- **聊天模型**：`MiniMax-M2.1` 或其他支持模型
- **向量数据库**：Chroma（本地持久化）
- **文本分割**：chunk_size=1000, chunk_overlap=100
